# 🧠 一、核心设计理念（最重要）

一句话总结：

> **把笔记当“代码工程”管理，而不是文档堆积**

对应三条原则：

### 1️⃣ 原子化（Atomic）

- 一条笔记 = 一个知识点
    
- 不写“大而全总结”
    

👉 类似函数：**单一职责**

---

### 2️⃣ 可链接（Linkable）

- 笔记之间必须有关系（双链）
    
- 用 Obsidian 的 `[[xxx]]`
    

👉 类似：

- 代码中的调用关系
    
- 模块依赖图
    

---

### 3️⃣ 可演化（Versioned）

- 用 Git 记录“认知变化”
    
- 不怕改，只怕混乱
    

👉 Git 的意义不是备份，而是：

- 看你是怎么“理解升级”的

# ✅ 方案二：Windows PowerShell

```powershell
$base = "knowledge-base"

"03-Areas",
"04-Resources",

}

New-Item -ItemType File -Path "$base\README.md"
```
