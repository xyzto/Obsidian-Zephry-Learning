# 知识管理方法论

> 关联领域：[[03-Areas/Zephyr-RTOS]] | 参考来源：多份 AI 整理文档提炼

## 一、核心理念

> **把笔记当成代码工程管理，而不是文档堆积。**

知识工程 = 问题驱动 + 结构化建模 + 实验验证 + 持续演化

---

## 二、三条黄金法则

### 1. 原子化 — 像写函数一样写笔记
一条笔记只讲一个知识点，不写大而全的总结。
- ❌ `STM32总结.md` → 知识垃圾堆
- ✅ `DMA.md` / `中断机制.md` / `USART.md` → 原子化，可复用

### 2. 可链接 — 像模块依赖一样建立关系
用 `[[...]]` 在笔记间建立双向链接，形成知识网络。每个 Concept 必须：
- ≥ 2 个 Concept 链接
- ≥ 1 个 Project 链接
- ≥ 1 个 Question 来源

### 3. 可演化 — 用 Git 记录认知升级
每次 commit 都是一次理解里程碑：
```bash
feat(concept): 建立DMA本质模型
fix(concept): 修正中断理解错误
test(project): 验证USART DMA接收
link(concept): 建立DMA与总线结构的关联
```

## 三、目录结构与角色

```
knowledge-base/
├── 00-Inbox/      # 捕获缓冲，不处理，48h 内清空
├── 01-Concepts/   # 知识原子，不允许直接写，必须经过 Questions
├── 02-Projects/   # 实验验证层，Projects = 验证工具，不是日志
├── 03-Areas/      # 领域导航，低频修改，只做目录
├── 04-Resources/  # 外部资料，只存引用，不写理解
├── 05-Questions/  # 问题驱动引擎，系统的核心入口
└── assets/        # 图片附件
```

## 四、唯一主流程

```
捕获(Inbox) → 提问(Questions) → 验证(Projects) → 沉淀(Concepts) → 连接([[链接]]) → 迭代(Git)
```

**强制规则：不允许直接写 Concepts，必须先经过 Questions。**

## 五、Question 笔记模板

```md
# 为什么 DMA 比中断快？

## 问题来源
项目：USART 接收实验

## 当前理解（必须先写猜测）
DMA 不需要 CPU 参与搬运

## 拆解
- 谁在搬数据？
- CPU 在做什么？
- 总线由谁控制？

## 验证
- [ ] 查阅参考手册
- [ ] 编写对比实验
- [ ] 用示波器/逻辑分析仪观察

## 结论
DMA 通过总线控制器直接在外设和内存间搬运数据，CPU 只需配置，无需参与每次传输。

## 关联
[[DMA]] | [[中断机制]] | [[USART]]

## 状态
- [x] 已验证
```

## 七、每日最小执行（4步）

1. **捕获**：看到不懂的 → 丢进 `00-Inbox`
2. **提问**：Inbox 内容 → 转化为 `05-Questions` 中的问题笔记
3. **验证**：查资料 / 写代码 / 做实验
4. **沉淀**：将结论写入 `01-Concepts`，建立链接，git commit

---

## 八、系统失效的三大原因

| 原因 | 表现 | 对策 |
|------|------|------|
| 写总结文档 | `STM32学习总结.md` | 强制原子化 |
| Inbox 堆积 | 超过一周未整理 | 每周固定 30 分钟清空 |
| 不建立链接 | 笔记孤立，Graph View 一片散点 | 每条 Concept 写完必须加链接 |

## 九、Obsidian + Git 快速启动

```bash
# Linux / Mac / WSL
mkdir -p brain/{00-Inbox,01-Concepts,02-Projects,03-Areas,04-Resources,05-Questions,assets}
cd brain && git init
echo ".obsidian/" > .gitignore
git add . && git commit -m "init: 建立知识库基础结构"
```

```powershell
# Windows PowerShell
$base = "brain"
@("00-Inbox","01-Concepts","02-Projects","03-Areas","04-Resources","05-Questions","assets") |
  ForEach-Object { New-Item -ItemType Directory -Path "$base\$_" }
cd $base; git init
".obsidian/" | Out-File .gitignore
git add .; git commit -m "init: 建立知识库基础结构"
```
