## 核心规则（必须遵守）

**1. 流程规则**
- 不允许直接写 `01-Concepts`，必须先经过 `05-Questions`
- 学到新概念 → 打开对应 Concept 文件补充，或新建 Concept
- 遇到问题 → 新建 Question 文件，写下猜测和拆解，搞懂后填结论
- 没有日记文件，没有中间层

**2. 结构规则**
- `01-Concepts/` 扁平结构，禁止任何子文件夹
- 用 `[[链接]]` 建立概念关联，不用文件夹层级表示关系
- 每个 Concept 必须有 ≥2 个概念链接、≥1 个来源 Question 链接

**3. 文件定位规则**

| 文件类型 | 放在哪里 |
|---------|---------|
| 知识概念（DMA / Thread） | `01-Concepts/` |
| 实验记录、验证过程 | `02-Projects/` 的 `实验/` 子目录 |
| 学科总览、学习进度 | `03-Areas/` |
| 外部链接、参考资料 | `04-Resources/` |
| 尚未解决的问题 | `05-Questions/` |
| 操作手册、系统配置 | `_维护/` |
| 随手捕获、待整理 | `00-Inbox/`（48h 内清空） |

**4. 禁止行为**
- 禁止创建"学习总结"、"笔记汇总"类文件
- 禁止在 `01-Concepts/` 里建子文件夹
- 禁止把操作手册放进 `04-Resources/`
- 禁止修改 `_维护/` 里的文件（除非我明确要求）

## 你每次操作前应该做的事

1. 用 `list_directory` 扫描相关目录，确认文件实际存在
2. 用 `read_text_file` 读取文件内容，不要凭记忆假设内容
3. 执行操作前告诉我你打算做什么，我确认后再执行
4. 操作完成后告诉我做了哪些改动，以及建议的 git commit 命令
