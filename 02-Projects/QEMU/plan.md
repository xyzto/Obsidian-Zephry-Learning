# 学习计划 — QEMU 仿真实验

> 用途：用 QEMU 快速验证 Zephyr 概念和逻辑，不依赖真实硬件
> 原则：先在 QEMU 跑通逻辑，再移植到 F103ZE 验证硬件行为

---

## QEMU 适合验证什么

| 适合 | 不适合 |
|------|--------|
| 内核调度逻辑 | 外设时序精度 |
| 线程间通信机制 | 真实传感器数据 |
| 内存管理行为 | 硬件中断响应时间 |
| 驱动 API 基本用法 | 特定芯片寄存器行为 |
| 子系统逻辑（Shell / Logging） | 射频 / BLE 真实通信 |

---

## 第一层：内核机制（线程与同步）

> 目标：彻底掌握 Zephyr RTOS 的核心调度和通信原语

| 编号 | 实验 | 验证目标 | 状态 |
|------|------|---------|------|
| 01 | 线程基础 | 双线程并发、优先级抢占、上下文切换 | ✅ |
| 02 | 信号量 | 生产者-消费者模型、事件通知 | ✅ |
| 03 | Mutex | 共享资源保护、优先级继承 | ✅ |
| 04 | 消息队列 | 线程间数据传递、队列满/空处理 | ✅ |
| 05 | 定时器 | k_timer 周期触发、单次触发、停止 | 待做 |
| 06 | 工作队列 | k_work 延迟处理、k_work_delayable | 待做 |
| 07 | 事件标志 | k_event 多事件组合等待 | 待做 |
| 08 | 内存池 | k_mem_pool / k_heap 动态分配 | 待做 |
| 09 | 栈溢出检测 | CONFIG_STACK_SENTINEL 触发行为 | 待做 |
| 10 | 调度器综合 | 多优先级、时间片、协作式混合场景 | 待做 |

## 使用方式

完成一个实验后：
1. 在 `实验/` 目录新建记录文件，命名格式：`编号-实验名称.md`
2. 在 `进度.md` 里打勾
3. 把新理解补充进对应的 `01-Concepts/` 文件
