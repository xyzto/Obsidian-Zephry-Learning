# 学习计划 — QEMU 仿真实验

> 用途：在没有真实硬件的情况下，用 QEMU 快速验证 Zephyr 概念和逻辑
> 适合：新概念验证、算法测试、多线程逻辑调试

---

## 使用原则

QEMU 实验不追求"做完"，而是用来**快速验证一个想法**。

- 学了新的 API → 先在 QEMU 上跑通，再移到真实硬件验证
- 调试多线程逻辑 → QEMU 比真实硬件更方便观察输出
- 验证 Zephyr 内核行为 → QEMU 足够

**不适合用 QEMU 的情况：** 外设时序、硬件中断响应时间、真实传感器数据。

---

## 常用 QEMU 目标板

| 目标板 | 架构 | 适合验证 |
|--------|------|---------|
| `qemu_cortex_m3` | ARM Cortex-M3 | 最接近 STM32，通用 Zephyr 功能 |
| `qemu_cortex_m0` | ARM Cortex-M0 | 最小 ARM，资源受限场景 |
| `native_sim` | Linux 进程 | 最快，适合纯逻辑和算法测试 |
