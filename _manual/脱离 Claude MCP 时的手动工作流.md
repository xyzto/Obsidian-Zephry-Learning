# 脱离 Claude MCP 时的手动工作流

## 核心思路

学习 AI（网页版）负责**生成内容**，你负责**手动复制粘贴**到仓库文件，不需要 MCP。

---

## 第一步：粘贴提示词

打开任意网页 AI（ChatGPT / Gemini / Claude 网页版等），第一条消息粘贴 `study-context.md` 的"开始粘贴"到"结束粘贴"之间的全部内容。

> `study-context.md` 已经包含了格式规范，不需要额外粘贴 `format-rules.md`。

---

## 第二步：做实验、触发总结

实验跑完后，对 AI 说：

```
实验跑完了，帮我整理笔记关键信息。
```

AI 会给你整理出：目标 / 现象 / 结论 / 疑问 四项。

---

## 第三步：让 AI 生成最终文件内容

确认内容无误后，说：

```
生成最终版
```

AI 会按格式规范输出一段纯净的 Markdown，没有任何多余文字，**直接可以复制**。

---

## 第四步：手动写入仓库

**实验记录：**

1. 在 `02-Projects\QEMU\lab\` 新建文件，文件名格式参考已有文件（如 `Exp06-Timer.md`）
2. 把 AI 输出的内容粘贴进去，保存

**progress.md 打勾：**

1. 打开 `02-Projects\QEMU\progress.md`
2. 找到对应实验，把 `待做` 改成 `✅`

**有坑的话补进 Concept：**

1. 打开 `01-Concepts\` 里对应的概念文件
2. 找到 `## 坑`，按格式追加一条

---

## 第五步：Git 提交

```bash
cd E:\OB-Vaults\Zephry
git add .
git commit -m "test(project): 完成实验 06 定时器"
```

---

## 总结对比

| 有 Claude MCP | 无 MCP（手动） |
|---|---|
| AI 直接写文件 | AI 生成内容，你复制粘贴 |
| 自动打勾 progress.md | 手动打勾 |
| 自动更新 session-log | 不需要（回来后 handoff 接续） |
| 全程对话完成 | 多 2 分钟手动操作 |

核心流程完全一样，只是最后一步从"AI 写文件"变成"你粘贴文件"，格式不会有任何差异。
