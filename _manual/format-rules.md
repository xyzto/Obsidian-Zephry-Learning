## 格式要点

**代码块语言标识符（必须严格遵守）**

| 代码类型 | 开头必须写 | 禁止写法 |
|---------|-----------|---------|
| C 代码（main.c） | ` ```c ` | ` ``` ` 或前面单独一行写 `C` |
| Kconfig 配置（prj.conf） | ` ```ini ` | ` ``` ` 或前面单独一行写 `Ini` / `TOML` |
| 设备树（app.overlay） | ` ```dts ` | ` ``` ` 或前面单独一行写 `DTS` |
| Shell 命令 | ` ```bash ` | ` ``` ` 或前面单独一行写 `Bash` |

**两种错误都会导致 Obsidian 无语法高亮：**
1. 代码块没有语言标识符：` ``` ` 后直接换行写代码
2. 语言名称写成独立行：在 ` ``` ` 前面单独一行写 `C`、`Bash`、`Ini, TOML` 等

**坑**
- 固定"现象/原因/解决"三行纯文本，不加任何 `#` 标题符号
- 多条坑之间用 `---` 分隔

**疑问与解答**
- Q 和 A 之间空一行，每组问答之间用 `---` 分隔
- 疑问是实验过程中真实产生的问题，不是泛泛的面试题

**现象部分**
- 用文字描述，不用有序列表（`1. 2. 3.`）

**链接路径**
- 反哺：`[[01-Concepts/概念名]]`，不得省略文件夹前缀
- 产生的问题：`[[05-Questions/问题名]]`，不得省略文件夹前缀

## 纯净输出规则

当用户说"生成最终版"、"给我最终内容"、"可以直接用的版本"时：
- 直接输出文件内容，不加任何前言、解释、说明
- 不写"以下是..."、"你可以..."之类的话
- 输出的内容复制后可以直接粘贴进 Obsidian，不需要二次删改
