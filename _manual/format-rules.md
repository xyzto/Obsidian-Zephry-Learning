## 格式要点

**代码块（最容易出错，严格遵守）**
- 语言标识符必须紧跟在三个反引号后面：` ```c `、` ```ini `、` ```bash `、` ```dts `
- **禁止**在代码块前单独写一行语言名称（如 `C`、`Bash`、`Ini, TOML`）——这会导致 Obsidian 无法识别语法高亮
- 正确示例：
  ````
  ```c
  int main(void) { return 0; }
  ```
  ````
- 错误示例：
  ````
  C
  ```
  int main(void) { return 0; }
  ```
  ````

**坑**
- 固定"现象/原因/解决"三行，**不加任何 `#` 标题符号**
- 多条坑之间用 `---` 分隔

**疑问与解答**
- Q 和 A 之间空一行
- 每组问答之间用 `---` 分隔
- 疑问是实验过程中真实产生的问题，不是泛泛的面试题

**链接路径**
- 反哺链接必须写完整路径：`[[01-Concepts/概念名]]`
- 产生的问题链接必须写完整路径：`[[05-Questions/问题名]]`
- 不得省略文件夹前缀

**现象部分**
- 用文字描述，不用有序列表（`1. 2. 3.`）

## 纯净输出规则

当用户说"生成最终版"、"给我最终内容"、"可以直接用的版本"或类似表述时：
- 直接输出文件内容，不加任何前言、解释、说明
- 不写"以下是..."、"你可以..."、"这是我为你生成的..."之类的话
- 不在内容前后附加任何 AI 对话文字
- 输出的内容复制后可以直接粘贴进 Obsidian，不需要二次删改
