# STM32 知识图谱

> 长期领域：嵌入式系统 | 状态：规划中，待 Zephyr 基础完成后建设

## 建模目标

```
STM32知识 = 外设机制 + 数据流 + 控制流 + 时序
```

核心问题：**数据是怎么在芯片里流动的？谁在控制？什么时候发生？**

---

## 待建立的 Concepts

**第一层（主干）**
- CPU / 总线结构 / 中断机制 / DMA / 时钟系统 / GPIO / USART / 定时器

**第二层（机制拆解）**
- NVIC / 中断优先级 / 中断向量表
- AHB总线 / APB总线
- DMA通道 / DMA触发源

**第三层（抽象概念）**
- 同步vs异步 / 阻塞vs非阻塞 / 轮询vs中断vsDMA / 数据搬运模型

---

## 待提出的 Questions

- 为什么 DMA 比中断快？
- DMA 什么时候触发？会不会占用总线？
- USART 为什么会丢数据？
- 数据从 USART 到内存经历了什么？
- CPU、DMA、总线三者关系是什么？

## 实验路线（02-Projects 规划）

| 阶段 | 实验 | 核心学习 |
|------|------|---------|
| 第一阶段 | GPIO 点灯 / USART 轮询发送 | 基础数据流 |

## 核心结构图

```
        +--------+
        |  CPU   |
        +--------+
             |
      +------+------+
      |   总线结构   |
      +------+------+
             |
 +-----------+-----------+
 |                       |
+-----+            +----------+
| DMA |            | 外设(USART)|
+-----+            +----------+
 |                       |
 +---------> 内存 <------+
```

**这张图 = STM32 系统的本质：谁在搬数据，通过哪条路，到哪里去。**
