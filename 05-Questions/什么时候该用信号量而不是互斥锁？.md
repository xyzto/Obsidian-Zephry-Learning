# 什么时候该用信号量而不是互斥锁？

> 来源：[[01-Concepts/Semaphore]]
> 状态：#已验证
> 关联概念：[[01-Concepts/Semaphore]] | [[01-Concepts/Mutex]]

## 当前猜测

信号量用于"通知"，Mutex 用于"保护"。区别在于有没有所有权。

## 拆解

- 信号量：无所有权，任何线程/ISR 都可以 give，用于协调顺序或计数资源
- Mutex：有所有权（谁锁谁解），用于保护临界区，有优先级继承

- [x] 阅读 Semaphore.md 和 Thread.md 中的对比表
- [x] 理解优先级继承机制（Priority_Inversion.md）

## 结论

**用信号量的场景**：
- 线程间事件通知（A 通知 B "可以干了"）：二值信号量，初始值 0
- 资源计数（限制同时访问 N 个资源）：计数信号量
- ISR → 线程的数据通知（ISR 只能 give，不能 lock Mutex）

**用 Mutex 的场景**：
- 保护共享资源临界区（全局变量、硬件外设）
- 需要优先级继承防止优先级翻转时
- 需要"谁锁谁解"的所有权语义时

**判断口诀**：问自己"我是在发信号，还是在抢地盘？"
- 发信号 → 信号量
- 抢地盘（保护资源） → Mutex

**禁止用信号量保护共享资源**：信号量无优先级继承，存在优先级翻转风险。
